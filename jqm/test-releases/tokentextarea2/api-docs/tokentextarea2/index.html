Token Text Area 2

This widget changes a text item to a button. It can consist of a number of button widgets. As the user types text, the widget parses the text at each keystroke and creates a button when the user types a character that acts as a delimiter (; or ,). A user can add the tokentextarea2 widget to a contact list, email list, or another list. The typical use of this widget is composing a number of contacts or phone numbers in a specific area of the screen.

Structure

The widget derives from the jQuery Mobile textinput widget. It works on all input elements marked with the attribute data-role="tokentextarea2" except those that are turned into sliders. Buttons are created from its initial value upon instantiation and if any text is left over it is set as the value of the input. To create a tokentextarea2 widget, add an input element to your markup:

1
 
<input type="text" id="email-addresses" data-role="tokentextarea2">

Keyboard interaction

The user may type freely into the input. Whenever she types a delimiter character, the text is tokenized the same way as initially, resulting in new buttons being inserted immediately preceding the input.

When the user types Backspace the button nearest the input will be selected in preparation for removal under the following circumstances:

  • There is at least one button before the input.
  • Pressing Backspace has not resulted in a change in the input’s value. That is, the value of the input is equal to the copy maintained in the visibility-hidden span used for calculating the width of the input (see below).

Layout and styling

The buttons are placed side-by-side. They wrap to the next line. The input is placed after the last button and sized such that it fills up the rest of the line. As the user types, the widget ensures that the input has sufficient width for accommodating the text typed. It measures the width of the text by maintaining a copy of the input’s value in a visibility-hidden span. If the width of the text exceeds the width of the input, the input width is restored to the textinput widget’s default of 100%, causing it to wrap to the next line. The widget is styled with a text cursor, making it appear as a large input. Clicking anywhere inside the widget except on one of the buttons will cause the input to receive focus. This further establishes the widget’s appearance of being a large input.

Interacting with the buttons

In addition to being able to remove buttons by pressing Backspace, the user can also click on them. The first click on a button will select it, while the second click will remove it. It is possible to prevent selection of a button by attaching a handler to the widget’s select event and preventing default on the event object.

Whenever the user types, all buttons are deselected. The deselection is performed only after making sure that the selected button immediately preceding the input is removed in response to a Backspace keypress that does not modify the value of the input (as described above).

Synchronizing with another input

The basic tokentextarea2 widget does not progressively enhance an input. This means that you have to use different APIs to access the value typed in by the user depending on whether jQuery Mobile considers the browser it is running on to be grade A or grade B, or whether it avoids all enhancement by considering the browser to be grade C. In the case of grade A and grade B browsers, you must use the inputText() method to retrieve the correct value from the input, because its .val() method does not retrieve any of the buttons.

jQuery Mobile does not run on browsers that it detects to be grade C – in most cases these are older browsers. This means that an input marked for enhancement into a tokentextarea2 widget will not be enhanced. It will remain a plain text input. This, in turn, means that you must use the .val() method to retrieve the correct value from the input, because the inputText() method has not been defined on the element.

The master option procided by the sync extension, together with the convenience classes ui-tokentextarea2-master and ui-tokentextarea2-slave addresses this discrepancy. To progressively enhance an input into a tokentextarea2, proceed as follows:

  1. Add two inputs to your markup:
    1. The first input is the one that you will use in your application. It is the input that holds the actual value you wish to retrieve, including all the buttons. You must give this input an id attribute (for example, id="email-addresses").
    2. The second input is enhanced into a tokentextarea2 and has the data-master attribute with its value set to a selector for the first input (for example, data-master="#email-addresses").
  2. Add class ui-tokentextarea2-master to the first input’s data-wrapper-class attribute: data-wrapper-class="ui-tokentextarea2-master". This will ensure that, whenever the tokentextarea2 widget is created, the master will not be shown.
  3. Add class ui-tokentextarea2-slave to the second input’s class attribute: class="ui-tokentextarea2-slave. This will ensure that the second input will not be shown on grade C browsers, but will be shown whenever the tokentextarea2 widget is created.

The tokentextarea2 demos provide an example of using the master option and the convenience classes.

Grouping

The grouping extension allows you to display a summary when the widget does not have the focus. It provides the option description which contains a string to be displayed when the widget does not have the focus. The string may contain the pattern {0} which will be replaced by the number of buttons currently present.

The extension also provides the public method focusOut() for placing the widget into a state where it displays no input and no buttons, but only the summary.

To restore the widget to its default look, you can call the public method focusIn(). This method is called automatically whenever the widget receives focus.

Providing pre-rendered markup

You can improve the load time of your page by providing the markup that the tokentextarea2 widget would normally create during its initialization.

By providing this markup yourself, and by indicating that you have done so by setting the attribute data-enhanced="true", you instruct the tokentextarea2 widget to skip these DOM manipulations during instantiation and to assume that the required DOM structure is already present.

When you provide such pre-rendered markup you must also set all the classes that the framework would normally set, and you must also set all data attributes whose values differ from the default to indicate that the pre-rendered markup reflects the non-default value of the corresponding widget option.

The tokentextarea2 widget has the same structure as the jQuery Mobile textinput widget with the following additional elements:

  • In addition to the textinput classes, the wrapper also has the class ui-tokentextarea2 and stretched-input.
  • The initial input value must not contain any substrings that can be turned into buttons. If you wish the widget to initially contain buttons, place an anchor for each button before the input. The widget adds a data item to each anchor it creates at key “value”. Thus, you must also add the data-value attribute to the anchor to store the substring that the button represents. The button gets the following classes: ui-btn ui-corner-all ui-shadow ui-tokentextarea2-button ui-btn-inline.
  • The widget synchronizes the input’s value to an internal span. This allows it to measure the width of the text, which, in turn, allows it to calculate at every keystroke whether the text typed by the user has enough room on the current line to display without causing the input to scroll horizontally. You must also add such span to your initial markup: <span class="ui-tokentextarea2-input-shadow"></span>.
  • If the initial value of the input is such that portions of it cannot be converted to a button, then you must set those portions as the initial text for the span.
  • If your build includes the grouping extension, you must also add a span for the summary.
  • If your build includes the link extension and you wish to have a link button, add the class ui-input-has-clear to the wrapper div and create an anchor with the link as the last child of the wrapper. The link must have at least class ui-input-clear. It is recommended that you create a small circular button with only an icon and no text.

Options

disabledType: Boolean

Default: false
Disables the tokentextarea2 if set to true.

This option is also exposed as a data attribute: data-disabled="true".

Code examples:

Initialize the tokentextarea2 with the disabled option specified:

1
 
$( ".selector" ).tokentextarea2({ disabled: true });

Get or set the disabled option, after initialization:

1
2
3
4
5
 
// getter
var disabled = $( ".selector" ).tokentextarea2( "option", "disabled" );
 
// setter
$( ".selector" ).tokentextarea2( "option", "disabled", true );

enhancedType: Boolean

Default: false
Indicates that the markup necessary for a tokentextarea2 widget has been provided as part of the original markup.

This option is also exposed as a data attribute: data-enhanced="true".

Code examples:

linkType: Selector or Element or jQuery

Default: null
Indicates which element is to be used as the widget’s link button. Setting this option to a null value will remove the link button.

This option is provided by the widgets/tokentextarea2.link extension.

This option is also exposed as a data attribute: data-link="#id-of-link-button".

Multiple types supported:

  • Selector: Indicates that the link button is to be retrieved from the DOM and placed inside the tokentextarea2 widget.
  • Element: Provides the element to place inside the tokentextarea2 widget. The class ui-input-clear is added to the element.
  • jQuery: Provides a jQuery collection object containing elements to place inside the tokentextarea2 widget to be used as the link button. The class ui-input-clear is added to the elements in the collection. The widget is designed for a single link button element.

Code examples:

Initialize the tokentextarea2 with the link option specified:

1
 
$( ".selector" ).tokentextarea2({ link: #id-of-link-button });

Get or set the link option, after initialization:

1
2
3
4
5
 
// getter
var link = $( ".selector" ).tokentextarea2( "option", "link" );
 
// setter
$( ".selector" ).tokentextarea2( "option", "link", #id-of-link-button );

masterType:

Default: null
Indicates that the tokentextarea2 is to update the value of the element identified by the option whenever its own inputText() changes. It also indicates that the inputText() of the tokentextarea2 is to be updated whenever the value of the element identified by the option changes. The update is performed in response to a “change” event triggered on the element identified by the option.

This option is provided by the widgets/tokentextarea2.sync extension.

This option is also exposed as a data attribute: data-master="#id-of-master".

Code examples:

Initialize the tokentextarea2 with the master option specified:

1

// setter

Methods

add( value )Returns: jQuery (plugin only)

Insert a new button with the specified value immediately before the input.
  • value
    Type: String
Code examples:

Invoke the method:

1
 
$( ".selector" ).tokentextarea2( "add", "my.name@example.com" );

add( value, index )Returns: jQuery (plugin only)

Insert a new button with the specified value at the specified zero-based index among the existing tokens. The button at that index and all buttons following it will move up by one. If the index specified is out of bounds, the button will be inserted immediately before the input.
  • value
    Type: String
  • index
    Type: Number
Code examples:

Invoke the method:

1
 
$( ".selector" ).tokentextarea2( "add", "my.name@example.com", 12 );

destroy()Returns: jQuery (plugin only)

Removes the tokentextarea2 functionality completely. This will return the element back to its pre-init state.
  • This method does not accept any arguments.
Code examples:

Invoke the destroy method:

1
 
$( ".selector" ).tokentextarea2( "destroy" );

focusIn()Returns: jQuery (plugin only)

This method is provided by the widgets/tokentextarea2.grouping extension.

Restores the default look of the widget. The input and its buttons become visible and the summary is hidden.

  • This method does not accept any arguments.
Code examples:

Invoke the focusIn method:

1
 
$( ".selector" ).tokentextarea2( "focusIn" );

focusOut()Returns: jQuery (plugin only)

This method is provided by the widgets/tokentextarea2.grouping extension.

Hides the input and any buttons, and displays the summary.

  • This method does not accept any arguments.
Code examples:

Invoke the focusOut method:

1
 
$( ".selector" ).tokentextarea2( "focusOut" );

inputText()Returns: String

Gets the string from the widget. The string consists of the ;-separated list of button values, followed by the value of the input.
  • This signature does not accept any arguments.
Code examples:

Invoke the method:

1
 
$( ".selector" ).tokentextarea2( "inputText" );

inputText( text )Returns: jQuery (plugin only)

Sets the value of the widget. All existing buttons are removed, the argument is tokenized, new buttons are created, and the leftover string is set as the input’s value.
  • text
    Type: String
    The new value the widget is to have.
Code examples:

Invoke the method:

1
 
$( ".selector" ).tokentextarea2( "inputText", "my.name@example.com; your.name@" );
