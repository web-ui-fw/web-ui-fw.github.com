HSVPicker

The Hue/Saturation/Value picker displays three sliders – one for each channel. Whenever the user drags one of the sliders, the resulting colour is calculated and assigned to the widget’s “color” option. The “colorchanged” signal is emitted. In addition, if the widget is based on a form input, the “change” signal is triggered on the input.

1
 
<div data-role="hsvpicker"></div>

Options

colorType: String

Default: null
The color to display/modify.

Code examples:

Initialize the hsvpicker with the color option specified:

1
 
$( ".selector" ).hsvpicker({ color: #f7b29a });

Get or set the color option, after initialization:

1
2
3
4
5
 
// getter
var color = $( ".selector" ).hsvpicker( "option", "color" );
 
// setter
$( ".selector" ).hsvpicker( "option", "color", #f7b29a );

Methods

destroy()Returns: jQuery (plugin only)

Destroys a hsvpicker. This causes the DOM elemenent on which it is based to be restored to the state in which it was before the widget was created. Any DOM elements created as part of the widget will be destroyed.
  • This method does not accept any arguments.
Code examples:

Invoke the destroy method:

1
 
$( ".selector" ).hsvpicker( "destroy" );

refresh()Returns: jQuery (plugin only)

Re-reads the color value from the native element and ensures the sliders’ values reflect the color.
  • This method does not accept any arguments.
Code examples:

Invoke the refresh method:

1
 
$( ".selector" ).hsvpicker( "refresh" );

Events

colorchanged( event )Type: colorchanged

Triggered when the color changes.
  • event
    Type: Event

Note: The ui object is empty but included for consistency with other events.

Code examples:

Initialize the hsvpicker with the colorchanged callback specified:

1
2
3
 
$( ".selector" ).hsvpicker({
  colorchanged: function( event, ui ) {}
});

Bind an event listener to the colorchanged event:

1
 
$( ".selector" ).on( "colorchanged", function( event, ui ) {} );
